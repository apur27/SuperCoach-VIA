## Anti-pattern list (standing)

Existing (retained, all still warranted):
- Never trust an LLM sum; re-measure disputed numbers in pandas before acting.
- Never verify by exit code; re-read the file content after any write.
- Never `git add .`; stage by explicit allowlist.
- Never hand-edit anything under `data/` or a generated table body.
- Never let a Pass-1 PASS stand in for Pass-2 clearance.
- Never soften an upstream caveat when translating numbers into prose.
- Never run the refresh before round settlement (Tuesday 8 PM UTC).
- Never push to `main` from parallel agents; serialize through one committer.
- A prompt must not mandate computation the agent's toolset cannot execute. (07-07)
- Never embed a data-file schema in a prompt as code; state where to discover it. (07-07)
- A lesson about agent X's failure mode belongs in agent X's memory directory. (07-07)
- A human decision that overrides a memory must retire or rewrite that memory in the same cycle. (07-07)
- Never bake ephemeral round/date state into a skill or prompt file. (07-07)

Added this survey:
- **Never write "committed"/"shipped" into a record before the commit exists** — completion notes
  follow the commit hash, they do not anticipate it. (F1: experiment-log:100, pending-decisions:71)
- **A repair recipe that needs a human/network run must carry a named runner and a review-by date**,
  or it silently becomes permanent corruption. (F2: recipe written 07-09, unrun, window growing)
- **Verify a dataset exists in the shape a plan assumes before scheduling work on it** — a dependency
  on repaired data is not a dependency on *sufficient* data. (F3: lineups have no position field)
