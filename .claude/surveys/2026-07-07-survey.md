# Survey — 2026-07-07 — scope: DEEP (agent-architecture review: memory, context, coordination, prompt quality)

Commissioned review of all nine agents across four dimensions. Every finding below was
verified against file contents read this session, not from memory or prior surveys.

## Executive read

The council's gates are sound but three agents are running on prompts that don't match
their jobs: BriefBuilder has no Bash tool yet is required to compute means computationally
(forcing the exact in-token-arithmetic failure class §4.4 exists to prevent), QA's embedded
verification code addresses a JSON schema that does not exist on disk, and FootyStrategy's
prompt never mentions the placeholder-filling work that is its primary pipeline duty.
Memory is rich but mis-routed — DataSentinel's hard-won false-FAIL lessons live in Gaffer's
and Scientist's directories where DataSentinel never reads them, and Scientist carries two
directly contradictory NaN-handling memories after the 2026-07-07 human decision. The single
highest-leverage move is a one-day prompt-hygiene pass (owner: Gaffer) fixing the tool
scope, the QA schema, and the FootyStrategy contract.

## Ranked findings (grouped by agent)

### BriefBuilder

#### F1 — [Prompt/Context] BriefBuilder — No Bash tool, but required to compute means computationally
**Severity:** HIGH [class: 4 — LLM-judgement-as-gate]
**Observation:** `.claude/agents/BriefBuilder.md:7` frontmatter: `tools: Read, Grep, Glob, Write, Edit` — no Bash. Hard rule at line 70: "compute it from raw row values using skipna=True … Always print the raw values list as a scratchpad step before writing the number." CLAUDE.md's verification rule requires running venv Python. With no Bash, every season mean, last-5 mean, and H2H margin in every skeleton is computed by in-token arithmetic — the documented §4.4 failure class, at the source of every number in every brief.
**Impact:** Wrong skeleton numbers are generated systematically; DataSentinel Pass 1 becomes the only defence and every miss it makes ships a fabricated-adjacent figure. Re-run churn on Pass 1 FAILs costs a chain segment each time.
**Fix:** Add `Bash` to the frontmatter tools line and a rule that every derived stat comes from an executed venv-Python one-liner whose output is quoted before the value is written. Owner: Gaffer (prompt hygiene; one line).

#### F2 — [Coordination] BriefBuilder — `<!-- SCIENTIST REVIEW -->` markers have no consumer
**Severity:** HIGH [class: NEW]
**Observation:** BriefBuilder.md:67,151 instructs leaving `<!-- SCIENTIST REVIEW: ... -->` comments for bias-correction calls. `.claude/skills/council-brief.md` steps 1–8 never invoke Scientist; no step routes these markers. `docs/architecture.md` §3 stamp includes `Scientist@<sha>` yet the chain has no Scientist step.
**Impact:** Bias flags (|bias| > 0.5 per-team) ship unresolved inside HTML comments; the stamp attests a Scientist involvement that structurally never happens on the brief lane.
**Fix:** council-brief.md gains an explicit step (or a Pass-1 sub-step): "grep the skeleton for SCIENTIST REVIEW markers; if any, dispatch Scientist and record the resolution before FootyStrategy." Owner: Gaffer.

#### F3 — [Memory] BriefBuilder — memory violates its own decay rules; duplicate topics

### FootyStrategy

#### F6 — [Prompt/Coordination] FootyStrategy — prompt omits its primary pipeline job
**Severity:** HIGH
**Observation:** `grep -c "FOOTYSTRATEGY INSERT\|coaches-strategy-corner\|afl-insights" .claude/agents/FootyStrategy.md` → 0. The prompt's interaction model (lines 61–78) is Q&A-shaped (Scientist finding → chat response). But council-brief.md step 3 requires filling `<!-- FOOTYSTRATEGY INSERT -->` placeholders in a doc on disk, and architecture §3 says it owns the `afl-insights.md` weekly recap. ## Coordination contract template (for every agent prompt)

```markdown
## HANDOFF CONTRACT
- **Invoked by:** <agent/skill/user> at <chain step>
- **Receives:** <exact inputs: file paths, round number, cycle type, verdicts — nothing else>
- **Reads:** <files it opens itself; canonical configs by path>
- **Produces:** <artifact path + format, or verdict schema>
- **Hands off to:** <next agent> with <what exactly>
- **On failure:** <verdict/halt behaviour> — routed to <owning agent> by <who>
- **Never:** <top 3 scope violations for this seat>
```
Gaffer.md, QA.md, DataSentinel.md and the skills already carry most of this implicitly;
FootyStrategy.md and Scientist.md carry none of it. Adding the block is the cheapest way
to make every handoff auditable against a single shape.

## Anti-pattern list (standing — additions this survey)

- (existing eight retained — see 2026-07-03 survey)
- A prompt must not mandate computation the agent's toolset cannot execute — tool scope and hard rules are one contract. (F1)
- Never embed a data-file schema in a prompt as code; state where to discover it, or the prompt rots the first time the schema moves. (F11)
- A lesson about agent X's failure mode belongs in agent X's memory directory, whatever agent learned it. (F4)
- A human decision that overrides a memory must retire or rewrite that memory in the same cycle. (F16)
- Never bake ephemeral round/date state into a skill or prompt file. (F21, and the "MEMORY.md is currently empty" snapshots, F18)
