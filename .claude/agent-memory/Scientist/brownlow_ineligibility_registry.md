The 2026 Brownlow doc (`docs/afl-brownlow-2026.md`) is auto-regenerated by `update_team_analysis.py` between the markers `<!-- 2026-BROWNLOW-PREDICTOR-START -->` and `<!-- 2026-BROWNLOW-PREDICTOR-END -->`. Direct edits inside those markers are wiped on the next refresh.

**Why:** Under AFL rules any in-season Tribunal suspension makes a player ineligible to win the Brownlow Medal — that rule lives outside the stat-profile proxy and has to be flagged in display.

**How to apply:** To flag a new suspended player, edit `BROWNLOW_INELIGIBLE_2026` in `update_team_analysis.py` (next to `BROWNLOW_WEIGHTS`). Key = `player_stem` (filename minus `_performance_details.csv`), value = `{"reason": "...", "team": "..."}`. The renderer (`_build_brownlow_table_md`) appends `**[SUSPENDED — BROWNLOW INELIGIBLE]**` inline, and `_build_brownlow_ineligibility_note` adds a footnote listing all flagged top-15 players. The proxy ranking itself is unchanged — suspended players keep their stat-based rank because the proxy is a stat-profile model, not a vote forecaster.

Regeneration is idempotent: re-running `generate_brownlow_predictor(pd.DataFrame(), 2026, max_round)` + `replace_brownlow_predictor_section` produces byte-identical output (verified 2026-05-18).
