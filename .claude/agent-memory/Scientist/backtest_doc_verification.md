afl-backtest-2026.md has THREE independently-generated numeric sections that drift to different round windows after an incremental backtest:
1. **Per-round table** (between `<!-- 2026-BACKTEST-START/END -->`) — auto-regenerated by update_team_analysis.py from MERGED summary CSVs (dedup by round, keep-latest). Usually current.
2. **Cumulative summary** (pooled MAE/RMSE/bias) — was stale at R1-R12 while per-round table was R1-R13.
3. **Team-level bias table** — most prone to staleness; observed at a smaller window (n~207/team) than the current data (n~276/team at R13).
4. **Top-30 player table** — was already current at R1-R13.
5. **Notable-misses table** — qualitative, only went to R11; entries come from the run-time log so they can disagree with a later re-run CSV (e.g. R1 Nick Daicos pred 21 in doc vs 20 in 20260525 re-run).

**Canonical sources (verified 2026-06):**
- Per-round metrics: merge all `backtest_summary_*.csv`, dedup by (year,round) keep-latest. Matches doc per-round table exactly.
- Pooled cumulative + team bias: concat the LATEST `prediction_vs_actual_round_<N>_2026_*.csv` per round, dropna on predicted+actual, compute on the pool.

**The 4,486 vs 4,806 reconciliation:** summary `n_players` is computed AFTER dropping rows with NaN actual_disposals (late-outs / non-played). Raw CSV rows R1-R13 = 4,815; 9 have NaN actual; scored = 4,806 = exact sum of summary n_players. 4,486 was the correct R1-R12 scored count; 4,806 is R1-R13.

**Overall (mean-of-round, R1-R13):** MAE 4.04, w5 73.0%, w10 95.7%, median round MAE 4.05.
**Pooled (player-weighted, R1-R13):** MAE 4.020, RMSE 5.155, bias -0.097, n=4,806.

Per-round prediction_vs_actual CSV row counts are SLIGHTLY higher than summary n (e.g. R3 csv=321 vs n=320) because of the NaN-actual rows. Always dropna before pooling.
