**Convention:** The user (Richmond-leaning AFL analyst working in the SuperCoach-VIA repo) runs post-match analyses as a structured brainstorm across N tactical themes (typically 6), each requiring: strategic read, what-it-means-for-each-team forward, and one bullet for "next time." Output is consumed downstream as repo documentation in post-match analysis files.

**Why:** Output format is fixed because it slots into a documentation pipeline; the user has stated the output is used to update post-match analysis sections in the repo.

**How to apply:** When a request arrives shaped as "brainstorm across N themes for [match]," do not collapse into a single integrated recommendation - preserve the per-theme structure. Apply the council deliberation protocol within each theme, but write the output in the user's themed-brainstorm format with sharp headers and a per-theme "what this means next time" bullet. Match the length the user requested (3-5 sentences per theme); do not over-expand.

**Vocabulary note:** User is comfortable with full AFL coaches'-box vocabulary (handball receiver, supply chain, KPF, half-back rebound, premiership quarter, etc). Do not dilute.
