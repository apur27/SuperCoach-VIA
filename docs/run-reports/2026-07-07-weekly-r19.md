## 6. Expansion Recommendations

**1. Make the eval-surface freshness honest, then make the backtest resumable.** (high impact / low effort — do first)
Stamp `afl-backtest-2026.md` from `max(round)` in the latest `backtest_summary_*.csv`, not from wall-clock date — a one-line change in the template that generates the "Last updated · N rounds backtested" line, plus a gate in `weekly_refresh.sh` that WARNs if `max_round_backtested < max_round_played`. **Why now:** this exact failure just shipped — a 2026-07-07 date over R17 content — and there is currently nothing that catches it.

**2. Add Optuna study persistence + `--resume` to `backtest.py`.** (high impact / medium effort)
Point the study at a SQLite storage (`optuna.create_study(storage=..., load_if_exists=True)`) so a killed run resumes from the last completed trial instead of restarting from zero. **Why now:** the run was killed at ~trial 48 and lost everything; this is the direct cause of the R18 gap and it will recur on any interrupted cycle.

**3. Ship prediction floor/ceiling (p10/p90) as the R19 board's next column.** (high impact / medium effort — endorses Surveyor S-5)
The prediction CSV is bare point estimates (`player, team, predicted_disposals` only), yet the backtest already computes `pct_within_5`/`pct_within_10`, so interval-coverage is measurable today. Train p10/p90 quantile companions and add `predicted_floor`/`predicted_ceiling`. **Why now:** the R19 top-eight are packed inside a 2-disposal band — point estimates can't separate them, but a ceiling can; and the coverage-validation harness to prove the intervals are calibrated already exists.

*Independent note vs. Surveyor backlog:* I rank the eval-surface honesty + backtest-resume pair (my #1/#2) **above** the product-feature items (S-4 fantasy, S-7 age features) that the Surveyor sequences highest, because this run proved the evaluation layer is silently fragile — new features are only as trustworthy as the backtest that validates them, and that backtest just failed to run.
