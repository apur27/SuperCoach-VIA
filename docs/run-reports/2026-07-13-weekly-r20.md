## 6. Expansion Recommendations

**1. Bring the HOF hub under a deterministic number gate — this cycle it blocked an otherwise-clean ship.**
Add `hall-of-fame-stat-leaders.md` to `scripts/check_hof_numbers.py`: validate each `<!-- HOF-HUB:career_* -->` sentinel cell against the JSON rank-1 total (the disposals cell already does — line 32 reads 11,088 correctly), and fail on the hand-written `[data]` prose lines (40, 66) that no gate re-derives. Pair with closing the kicks/handballs two-leader regeneration so line 66 is machine-written, not hand-typed. **Impact:** removes the "held-back hub" from becoming a standing weekly exception; closes a silent-drift path. **Effort:** 2–3 hours. **Why now:** this run is the first where the drift was caught *before* ship rather than after — the fix converts a manual catch into a permanent gate.

**2. Add a per-team bias gate to `weekly_refresh.sh` keyed to actual disposal scale (not a >20 placeholder).**
`backtest_by_team_<ts>.csv` is now generated every week (three consecutive files present). Add `scripts/check_prediction_bias.py` that fails if any team's per-round MAE exceeds ~1.6× the global MAE (≈6.3 this week) **or** absolute bias exceeds 2.5 disposals. This round it would flag St Kilda (bias −2.83) for review — a real, actionable signal. **Impact:** catches systematic per-team drift before it reaches predictions and briefs. **Effort:** 1–2 hours. **Why now:** the by-team artifact is stable and three weeks deep; the disposal-scale thresholds (single-digit MAE, ~2× best-to-worst spread) are now empirically known, so the gate can be calibrated to reality rather than guessed.

**3. Ship prediction intervals — the eval now proves the coverage to size them honestly.**
The prediction CSV is still bare `player, team, predicted_disposals`. The R1–R19 eval gives calibrated empirical coverage (within-5 74.2%, within-10 95.8%) and a known tier effect (top-10 MAE ~2.5× global). Add `predicted_low`/`predicted_high` columns — a tier-conditional empirical band (e.g. ±5 baseline, wider for elite-tier players) — and extend `backtest.py` to report interval coverage alongside MAE. **Impact:** turns a point estimate into a usable range for fantasy/betting readers and makes the elite-tier weakness legible instead of hidden. **Effort:** ~1 day. **Why now:** the walk-forward harness has enough scored rounds (6,792 player-rounds) to set band widths from observed error, not assumption.
